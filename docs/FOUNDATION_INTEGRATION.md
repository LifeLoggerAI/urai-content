# URAI Foundation Integration

URAI Content uses URAI Foundation as the public-claims, accessibility, provenance, consent, and accountability anchor for content systems and published materials.

## Canonical references

- Foundation repository: `LifeLoggerAI/urai-foundation`
- Public domain target: `https://uraifoundation.org/`
- Governance charter: `https://uraifoundation.org/docs/governance-charter.md`
- Ethical AI principles: `https://uraifoundation.org/docs/ethical-ai-principles.md`
- Transparency framework: `https://uraifoundation.org/docs/transparency-framework.md`
- Risk review process: `https://uraifoundation.org/docs/risk-review-process.md`
- System-of-systems contract: `https://uraifoundation.org/docs/system-of-systems-integration.md`

## Content alignment requirements

Content changes should reference Foundation standards when they affect:

- public claims about AI, privacy, safety, governance, accessibility, or user benefit;
- educational or research-facing materials;
- provenance, attribution, licensing, or ownership claims;
- sensitive user stories, emotional context, relationship context, health-adjacent content, or vulnerable communities;
- launch copy, public docs, trust pages, or partner collateral.

## Release gate

For major public content releases, review language against Foundation ethical AI principles and the transparency framework.

## Live-domain caveat

Do not treat `uraifoundation.org` as live on GitHub Pages until DNS no longer resolves to Squarespace and `/sitemap.xml` returns `200`.
