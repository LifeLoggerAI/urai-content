# Launch Asset Checklist

## Public experience

- Landing page
- Founder page
- Technology page
- Ecosystem page
- Receipts page
- Privacy page
- Press page
- Contact page

## Visual assets

- Product screenshots
- Spatial world screenshots
- Demo walkthrough
- Architecture diagram
- System map
- Founder image/profile assets
- Press kit graphics

## Evidence assets

- Repository links
- Release notes
- Build receipts
- Test evidence
- Deployment evidence
- Privacy review evidence
- Known limitations

## Trust assets

- Clear company description
- Clear product description
- Responsible claims
- Contact path
- Terms and privacy links

## Final launch rule

Every public statement should map to either a live demo, repository evidence, or documented receipt.
